# 手牌管理系统

> 状态：Draft
> 层级：Core
> 优先级：MVP

## 概述

管理玩家的手牌区域——最多持有 3 张干支牌。手牌是玩家与季节系统交互的载体，持仓收益和卖出操作都基于手牌中的牌。格子的稀缺性迫使玩家做出取舍。

## 玩家体验

玩家始终感受到"空间不够"的压力——想买的牌很多，但只能持有 3 张。每买入一张新牌，就要考虑该腾出哪个位置。手牌的每一张都在持续产生盈亏，空着虽然不亏但也不赚。

## 详细规则

### 手牌区域

| 属性 | 规则 |
|------|------|
| 最大容量 | 3 个格子 |
| 初始状态 | 空（0 张牌） |
| 最小容量 | 0 张牌 |

### 买入操作

```
前提条件：
  1. 手牌 < 3 张（有空位）
  2. 气足够支付买入消耗

执行：
  1. 从公共展示区选择 1 张牌
  2. 选择是否使用杠杆（显示当前季节杠杆倍数）
  3. 扣除对应气量（杠杆买入额外消耗 LQC=10 气）
  4. 牌进入手牌（记录：杠杆倍数、买入时评分）
  5. 未选的牌回到牌堆随机位置
  6. 从本回合起开始计算持仓收益（杠杆牌 × 杠杆倍数）
  7. 每回合扣除持仓气耗 = HOLD_QI_COST × 杠杆倍数
```

### 卖出操作

```
前提条件：
  1. 手牌 ≥ 1 张
  2. 气足够支付卖出消耗

执行：
  1. 从手牌中选择 1 张牌
  2. 扣除卖出气量（3气）
  3. 计算卖出得分（杠杆牌收益 × 杠杆倍数），累加总分
  4. 回复 CS(8) 气
  5. 牌永久移除（不回牌堆）
  6. 腾出 1 个空位
```

### 持仓结算

```
每回合开始时（季节检查之后、抽牌之前）：
  对手牌中每张牌：
    得分 += 盈亏倍率(牌的主属性, 当前季节) × 基础分值
    总分 += 得分
```

> 注意：持仓收益是自动结算的，不需要玩家操作。即使玩家选择"等待"，持仓牌依然产生盈亏。

### 手牌满时的限制

| 操作 | 手牌满时（3 张） |
|------|-----------------|
| 买入 | ❌ 不可执行，提示"手牌已满" |
| 卖出 | ✅ 正常执行，腾出空位 |
| 等待 | ✅ 正常执行，回气 |

> 玩家必须先卖出才能买入新手牌。卖出不是免费的（消耗气），所以"换牌"的代价是双倍气消耗（卖出 + 买入）。

### 空格子的含义

- 空格子不产生任何收益
- 空格子也不产生任何亏损
- 持有空位 = 浪费了潜在的持仓收益
- 但持有亏损牌 = 每回合都在扣分
- 有时"空着"比"亏着"好——这就是决策

### 卖出后牌的去向

- 卖出的牌**永久移除**，不回牌堆，不可再被任何人获取
- 卖出的牌记录在历史日志中（可选功能）

## 公式

### 手牌持仓每回合总分

```typescript
function calculateHoldScore(hand: JiaziCard[], season: Season): number {
  let totalScore = 0;
  for (const card of hand) {
    const multiplier = getSeasonMultiplier(card.mainElement, season);
    totalScore += multiplier * BASE_SCORE;
  }
  return totalScore;
}
```

### 卖出一次性得分

```typescript
function calculateSellScore(card: JiaziCard, season: Season): number {
  const multiplier = getSellMultiplier(card.mainElement, season);
  return multiplier * BASE_SCORE;
}
```

## 边界情况

| 场景 | 处理方式 |
|------|----------|
| 手牌为空且选择卖出 | 提示"没有可卖出的牌"，操作无效 |
| 手牌 3 张且想买入 | 提示"手牌已满"，必须先卖出 |
| 持有 3 张对立季节牌 | 每回合大量扣分，应尽快卖出止损 |
| 卖出后立刻买入同回合 | ❌ 不允许，每回合只能选一个操作 |
| 季节切换时持仓牌盈亏变化 | 正常，下回合按新季节结算 |
| 持有土牌 + 其他季节牌 | 土牌稳定收益，其他牌按季节涨跌 |
| 手牌 0 张全程等待 | 只靠气的回复，但没有任何得分来源 |

## 依赖关系

| 系统 | 方向 | 性质 |
|------|------|------|
| 干支卡牌数据 | 依赖 | 读取牌的属性信息 |
| 季节循环 | 依赖 | 读取当前季节计算盈亏 |
| 气资源 | 被依赖 | 消耗气执行买入/卖出 |
| 公共牌池 | 依赖 | 从展示区获取牌 |
| 盈亏结算 | 被依赖 | 提供手牌列表进行结算 |
| UI 渲染 | 被依赖 | 展示手牌区域、盈亏提示 |

## 调参旋钮

| 参数 | 默认值 | 安全范围 | 影响 |
|------|--------|----------|------|
| 手牌上限 | 3 | 2 ~ 5 | 越小越紧张，越大策略空间越宽 |
| 持仓收益起始回合 | 买入当回合 | 买入当回合 / 下一回合 | 当回合生效 vs 下一回合生效 |

### 扩展预留

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 是否允许手牌排序/交换位置 | false | 纯展示用途，不影响逻辑 |
| 卖出历史记录 | false | Alpha 阶段可开启 |

## 验收标准

- [ ] 手牌上限 3 张，不可超出
- [ ] 买入正确添加牌到手牌空位
- [ ] 卖出正确移除牌并腾出空位
- [ ] 手牌满时买入被阻止
- [ ] 手牌空时卖出被阻止
- [ ] 持仓收益每回合自动结算
- [ ] 卖出得分一次性结算并累加总分
- [ ] 卖出的牌永久移除不回牌堆
- [ ] 每回合只能执行买入或卖出或等待，不可组合操作
