# 回合流程系统

> 状态：Draft
> 层级：Core
> 优先级：MVP

## 概述

游戏的主循环引擎，将所有子系统按正确顺序串联起来。每回合执行固定的步骤流程，管理游戏状态流转，是整个游戏的骨架。

## 玩家体验

玩家感受到清晰的节奏感——每回合都是"看分数变化 → 看新牌 → 做决策"的三步循环。没有任何模糊地带，该做什么、能做什么，一目了然。

## 详细规则

### 游戏初始化

```
1. 生成 60 甲子牌数据，洗牌形成牌堆
2. 随机生成四季长度（每季 3-12 回合）
3. 初始化玩家状态：
   - 气 = 50
   - 总分 = 0
   - 手牌 = 空（3 个空位）
   - 当前季节 = 春
   - 当前回合 = 1
   - 季节内回合 = 1
```

### 单回合流程

```
回合 N 开始
  │
  ├─ 1. 游戏结束检查
  │     ├─ 如果 当前回合 > 60：
  │     │     进入游戏结束流程
  │     └─ 否则：继续
  │
  ├─ 2. 季节检查
  │     ├─ 如果 季节内回合 > 本季长度：
  │     │     切换到下一季节（春→夏→秋→冬→春）
  │     │     季节内回合 = 1
  │     │     提示玩家"季节已切换"
  │     └─ 否则：
  │           继续（季节内回合 += 1 在回合推进时执行）
  │
  ├─ 3. 持仓结算 + 持仓气耗扣除
  │     ├─ 对手牌中每张牌计算持仓收益：
  │     │     本回合盈亏 = HOLD_BONUS(1.2) × 卡牌评分(卡牌, 当前季节) × 杠杆倍数
  │     │     累加到总分
  │     ├─ 扣除持仓气耗（评分驱动：max(0.5, 1.5 + 0.4 × 当季评分) × 杠杆倍数）
  │     ├─ 气归零时触发爆仓（随机强平一张杠杆牌）
  │     └─ 更新 UI（分数变化、盈亏明细）
  │
  ├─ 4. 刷牌
  │     ├─ 从牌堆抽取 min(2, 牌堆剩余) 张牌
  │     ├─ 展示在公共牌池区域
  │     └─ 如果牌堆为空：公共牌池显示"牌堆已空"
  │
  ├─ 5. 气回复
  │     ├─ 每回合自然回复 QR(7) 气
  │     └─ 如果 上一回合操作 == "等待"：
  │           气 = min(80, 气 + WR(10))
  │
  ├─ 6. 等待玩家操作
  │     ├─ 可选操作：买入 / 卖出 / 等待
  │     │
  │     ├─ 买入：
  │     │     ├─ 前提：手牌 < 3 且 气 >= 买入消耗
  │     │     ├─ 从公共牌池选 1 张
  │     │     ├─ 选择是否使用杠杆（显示当前杠杆倍数）
  │     │     ├─ 扣气（消耗 = CB(12) × (1 + 0.05 × 评分) [+ LQC(10) 如使用杠杆]）
  │     │     ├─ 牌进入手牌（记录杠杆倍数和买入评分）
  │     │     ├─ 未选的牌回牌堆随机位置
  │     │     └─ 记录本回合操作 = "买入"
  │     │
  │     ├─ 卖出：
  │     │     ├─ 前提：手牌 ≥ 1 且 气 >= 卖出消耗
  │     │     ├─ 从手牌选 1 张
  │     │     ├─ 扣气（3气）
  │     │     ├─ 计算卖出得分：(SB(8) + (卖出评分 - 买入评分) × SM(4)) × 杠杆倍数
  │     │     ├─ 即时回复 CS(8) 气
  │     │     ├─ 牌永久移除
  │     │     └─ 记录本回合操作 = "卖出"
  │     │
  │     └─ 等待：
  │           ├─ 公共牌回牌堆随机位置
  │           ├─ 无气消耗
  │           └─ 记录本回合操作 = "等待"
  │
  ├─ 7. 推进回合
  │     ├─ 当前回合 += 1
  │     ├─ 季节内回合 += 1
  │     └─ 回到回合 N+1
```

### 游戏结束流程

```
1. 最后一回合（第 60 回合）正常执行完整回合流程
2. 回合结束后进入结算界面：
   ├─ 显示最终总分
   ├─ 显示得分构成：
   │     ├─ 总持仓收益
   │     ├─ 总卖出收益
   │     └─ 总买入消耗（气的使用统计）
   ├─ 显示游戏统计：
   │     ├─ 总买入次数
   │     ├─ 总卖出次数
   │     ├─ 总等待次数
   │     ├─ 总操作次数
   │     └─ 气利用率（消耗总气 / 理论可用总气）
   └─ 按钮：再来一局 / 返回主页
```

### 状态机

```
[游戏初始化] → [回合进行中] → [等待玩家操作] → [回合结算]
     ↑                                              │
     └──────────────────────────────────────────────┘
                                    │
                                    ↓ (回合 > 60)
                              [游戏结束]
                                    │
                                    ↓
                              [再来一局] → [游戏初始化]
```

### 游戏状态快照

```typescript
interface GameState {
  // 基础信息
  round: number;              // 当前回合 1-60
  season: Season;             // 当前季节
  seasonRound: number;        // 当前季节内第几回合
  seasonLength: number;       // 当前季节总回合数
  seasonIndex: number;        // 当前处于第几个季节段
  seasonLengths: number[];    // 所有季节的长度 [春, 夏, 秋, 冬, ...]

  // 牌堆
  deck: JiaziCard[];          // 剩余牌堆
  deckSize: number;           // 剩余牌数（展示用）

  // 公共牌池
  publicCards: JiaziCard[];   // 当前展示的牌（0-2 张）

  // 玩家状态
  qi: number;                 // 当前气
  score: number;              // 总分
  hand: (HandSlot | null)[];  // 手牌（最多 3 个位置，HandSlot 含买入评分和杠杆信息）
  lastAction: 'buy' | 'sell' | 'wait' | null; // 上一回合操作
  phase: 'init' | 'season_check' | 'settlement' | 'draw' | 'qi_recover' | 'player_action' | 'ui_update' | 'game_over';

  // 统计
  stats: {
    totalBuys: number;
    totalSells: number;
    totalWaits: number;
    totalHoldEarnings: number;
    totalSellEarnings: number;
    totalQiSpent: number;
    totalQiRecovered: number;
    forcedSells: number;        // 爆仓强平次数
    totalLeverageBuys: number;  // 杠杆买入次数
  };
}

// 手牌项（含买入上下文）
interface HandSlot {
  card: JiaziCard;           // 卡牌数据
  buyScore: number;          // 买入时的评分
  leverage: number;          // 买入时杠杆倍数（1.0 = 无杠杆）
  buyRound: number;          // 买入时的回合数
  holdEarnings: number;      // 累计持有收益（每回合利息之和）
}
```

## 边界情况

| 场景 | 处理方式 |
|------|----------|
| 季节切换恰好是第 60 回合 | 正常切换，按新季节结算第 60 回合持仓收益 |
| 牌堆已空且手牌为空 | 玩家只能等待到游戏结束，无法做任何操作 |
| 牌堆已空但手牌有牌 | 玩家可以选择卖出或等待持仓收益 |
| 玩家气为 0 且无法等待回复 | 理论上不会发生（等待不消耗气），但如果卖出也买不起，只能等 |
| 游戏开始第一回合 | 没有持仓收益（手牌为空），直接进入刷牌阶段 |
| 玩家所有操作都选等待 | 气一直满，但分数永远为 0 |
| 季节长度随机导致总回合不足 60 | 按需追加季节循环，确保 60 回合填满 |

### 季节长度与 60 回合的匹配

```
问题：4 个季节随机 3-12 回合，总和不一定是 60

策略：预生成足够多的季节长度，填满 60 回合

示例：
  生成 [5, 8, 4, 10, 7, 6, 9, 3, 8] = 60
  即：春(5)→夏(8)→秋(4)→冬(10)→春(7)→夏(6)→秋(9)→冬(3)→春(8，只打 8 回合就结束)

最后一个季节可能不完整（只打部分回合）
```

## 依赖关系

| 系统 | 方向 | 性质 |
|------|------|------|
| 干支卡牌数据 | 依赖 | 初始化牌堆 |
| 季节循环 | 依赖 | 季节检查与切换 |
| 气资源 | 依赖 | 气回复与消耗 |
| 公共牌池 | 依赖 | 刷牌 |
| 手牌管理 | 依赖 | 买入/卖出操作 |
| 盈亏结算 | 依赖 | 持仓结算 |
| UI 渲染 | 被依赖 | 触发 UI 更新 |

## 调参旋钮

| 参数 | 默认值 | 安全范围 | 影响 |
|------|--------|----------|------|
| 游戏总回合数 | 60 | 40 ~ 100 | 游戏时长 |
| 季节长度范围 | 3-12 | 2-15 | 不确定性程度 |

## 验收标准

- [ ] 游戏初始化正确（牌堆洗牌、季节生成、玩家状态）
- [ ] 每回合按固定顺序执行 7 个步骤
- [ ] 游戏结束检查在季节检查之前
- [ ] 持仓结算包含评分驱动的动态气耗扣除
- [ ] 爆仓检查紧跟持仓气耗扣除
- [ ] 气回复基于上一回合操作判断（卖出不产生额外下回合回复，而是在卖出时即时回复 8 气）
- [ ] 玩家只能选买入/卖出/等待三选一
- [ ] 买入/卖出前提条件正确检查
- [ ] 等待时公共牌回牌堆
- [ ] 60 回合后正确进入游戏结束
- [ ] 季节长度随机生成，总和覆盖 60 回合
- [ ] 游戏状态完整保存，可正确重置
- [ ] 统计数据正确累计（含爆仓强平次数、杠杆买入次数）
