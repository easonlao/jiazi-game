# 公共牌池系统

> 状态：Draft
> 层级：Core
> 优先级：MVP

## 概述

管理游戏中的牌堆和公共展示区。每回合从牌堆抽取 2 张牌供玩家二选一，未选的牌回到牌堆随机位置。牌池是玩家获取卡牌的唯一途径，资源有限且不可再生。

## 玩家体验

玩家每回合看到 2 张牌，需要快速判断：这张牌值不值得花气买？不买的话它就消失在牌堆里，以后可能再也见不到。资源是有限的，错过可能就没有了。

## 详细规则

### 牌堆结构

| 项目 | 规则 |
|------|------|
| 总牌数 | 60 甲子牌（MVP 阶段不含特殊牌） |
| 初始状态 | 60 张牌洗牌后形成牌堆 |
| 牌的流向 | 牌堆 → 公共展示 → 玩家手牌（买入）或 牌堆随机位置（未选） |
| 牌是否回归 | 买入后永久移除，不会重回牌堆 |
| 牌堆耗尽 | 60 张牌全部被抽完后，公共展示区不再刷牌 |

### 每回合流程

```
1. 从牌堆抽取 2 张牌 → 进入公共展示区
2. 玩家选择：
   a. 买入其中 1 张 → 消耗气，牌进入手牌
   b. 不买入 → 两张牌都回到牌堆随机位置
3. 如果只选了 1 张，另 1 张回到牌堆随机位置
```

### 回到牌堆的规则

- 未被选中的牌回到牌堆的**随机位置**
- 不会回到牌堆顶部（防止玩家预判下一回合的牌）
- 插入位置范围：牌堆的第 1 位到最后一位之间均匀随机

### 牌堆耗尽后的处理

| 阶段 | 行为 |
|------|------|
| 牌堆 ≥ 2 张 | 正常抽取 2 张 |
| 牌堆 = 1 张 | 抽取 1 张展示，玩家可选择买或不买 |
| 牌堆 = 0 张 | 公共展示区为空，玩家只能等待或卖出 |

> 牌堆耗尽后游戏进入后半段，玩家只能靠持仓收益和卖出来赚分，策略重心从"选牌"转向"择时卖出"。

### 牌的可预测性

- 玩家**无法查看**牌堆中剩余的具体牌
- 玩家**可以查看**牌堆剩余数量（"剩余 XX 张"）
- 玩家**可以查看**已出现的牌（历史记录，可选功能）
- 未选的牌回到随机位置，无法追踪位置

### 牌的抽取公平性

- 牌堆使用 Fisher-Yates 洗牌算法，确保完全随机
- 回到牌堆的牌插入位置使用均匀随机分布
- 无牌堆查看机制，杜绝"记牌"优势

## 公式

### 抽牌逻辑

```typescript
function drawCards(deck: JiaziCard[], count: number): { drawn: JiaziCard[], remaining: JiaziCard[] } {
  const actualCount = Math.min(count, deck.length);
  const drawn = deck.splice(0, actualCount);
  return { drawn, remaining: deck };
}
```

### 未选牌回堆

```typescript
function returnToDeck(card: JiaziCard, deck: JiaziCard[]): void {
  // 随机插入位置（0 到 deck.length 之间，均匀分布）
  const position = Math.floor(Math.random() * (deck.length + 1));
  deck.splice(position, 0, card);
}
```

## 边界情况

| 场景 | 处理方式 |
|------|----------|
| 牌堆只剩 1 张 | 只展示 1 张，玩家可买或不买 |
| 牌堆为空 | 不展示牌，玩家只能等待或卖出 |
| 玩家手牌已满（3 格）且想买入 | 无法买入，只能卖出或等待 |
| 玩家气不够买入任何展示牌 | 可以选择等待（牌回堆）或卖出 |
| 两张展示牌都很有吸引力 | 只能选 1 张，强制取舍 |
| 两张展示牌都不想要 | 都不买，回堆 |
| 同一张牌被多次抽到又多次回堆 | 正常现象，概率随机 |
| 游戏结束时牌堆还有剩余 | 正常，不影响结算 |

## 依赖关系

| 系统 | 方向 | 性质 |
|------|------|------|
| 干支卡牌数据 | 依赖 | 提供完整的 60 甲子牌数据 |
| 气资源 | 被依赖 | 判断玩家是否有足够气买入 |
| 手牌管理 | 被依赖 | 判断手牌是否已满 |
| 回合流程 | 被依赖 | 每回合触发抽牌 |
| UI 渲染 | 被依赖 | 展示公共牌池、牌堆剩余数量 |

## 调参旋钮

| 参数 | 默认值 | 安全范围 | 影响 |
|------|--------|----------|------|
| 每回合展示牌数 | 2 | 1 ~ 3 | 选择多=决策复杂，选择少=运气成分大 |
| 牌堆总量 | 60 | 固定 | 60 甲子，不可调 |

### 扩展预留

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 是否显示已出现的牌历史 | false | Alpha 阶段可开启 |
| 未选牌回堆位置策略 | 完全随机 | 未来可改为"偏向牌堆中部"等策略 |

## 验收标准

- [ ] 游戏开始时 60 张牌正确洗牌
- [ ] 每回合正确抽取 2 张牌展示
- [ ] 玩家只能选 1 张或都不选
- [ ] 未选的牌回到牌堆随机位置
- [ ] 买入的牌从牌堆永久移除
- [ ] 牌堆剩 1 张时只展示 1 张
- [ ] 牌堆为空时不再展示牌
- [ ] 牌堆剩余数量对玩家可见
- [ ] 洗牌和插入位置完全随机
