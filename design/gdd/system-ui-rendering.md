# UI 渲染系统

> 状态：Draft → Updated
> 层级：Presentation
> 优先级：MVP
> 更新记录：2026-04-27 补全杠杆交互、动画规范、爆仓反馈、屏幕适配

## 概述

游戏在手机竖屏上的完整界面布局与交互设计。风格为国风卡通，五行配色鲜明，操作简洁直观，适合单手操作。

## 玩家体验

玩家打开游戏就像打开一本精美的古籍——国风底色但不沉闷，卡通风格活泼有趣。所有信息一屏呈现，不需要滚动，拇指操作区域集中在屏幕下半部分。操作有动画反馈，每次买入卖出都能感受到"东西在动"。

## 详细规则

### 屏幕适配

| 属性 | 规则 |
|------|------|
| 设计基准 | 428×760（iPhone 14 Pro Max） |
| 最小支持宽度 | 375px（iPhone SE） |
| 缩放策略 | 以 428px 宽度为基准，等比缩放（`scale = screenWidth / 428`） |
| 安全区域 | 适配刘海屏、底部横条（CSS env / Phaser scale manager） |
| 横屏处理 | 检测横屏时显示"请旋转手机至竖屏"提示，锁定竖屏方向 |
| 实现方式 | Phaser Scale Manager: `mode: Phaser.Scale.FIT`, `autoCenter: Phaser.Scale.CENTER_BOTH` |

### 各区域详细设计

#### 1. 顶部信息栏

```
内容：季节图标 + 季节名 + 进度（仅显示当前第几回合）
示例：🌸 春 (第3回合)
```

| 元素 | 说明 |
|------|------|
| 季节图标 | 春🌸 夏☀️ 秋🍂 冬❄️（卡通风格） |
| 季节名称 | 春 / 夏 / 秋 / 冬 |
| 季节进度 | 当前第几回合（严禁显示总回合数以保持博弈感） |
| 季节进度预警 | 季节内回合 ≥ 8 时，季节名文字轻微闪烁（暗示换季临近） |
| 季节切换 | 全屏背景色渐变过渡（500ms） + 季节图标缩放弹入 |

#### 2. 分数区域

```
总分: 128           ▲ +5
```

| 元素 | 说明 |
|------|------|
| 总分 | 大字体，左对齐 |
| 上回合盈亏 | 右对齐，正数绿色带 ▲，负数红色带 ▼，零灰色，无数据时不显示 |
| 分数变化动画 | 总分数字缩放弹跳（scale 1.0→1.15→1.0，300ms） |

#### 3. 气资源条

```
████████░░░░  气 75/80
持仓维持: -6气/回合
```

| 元素 | 说明 |
|------|------|
| 气条 | 横向进度条，从左到右填充 |
| 气条颜色 | >60% 绿色，30-60% 黄色，<30% 红色 |
| 数值显示 | 当前气/最大气，白色文字居中 |
| 持仓维持 | 气条下方小字，显示每回合持仓总气耗（含杠杆加成），无持仓时不显示 |
| 气量变化动画 | 回复时气条绿色闪烁（alpha 1→0.5→1，200ms），消耗时红色闪烁 |
| 气量不足时 | 气条边框变红（描边），买入/卖出按钮置灰 |

#### 4. 公共牌池（展示区）

```
  ┌──────────┐  ┌──────────┐
  │   甲 子   │  │   丙 寅   │
  │  (阳木)   │  │  (阳火)   │
  │    木     │  │    火     │
  │  🟢+3.0  │  │  🔴-1.0  │
  │  消耗: 14 │  │  消耗: 10 │
  └──────────┘  └──────────┘
```

| 元素 | 说明 |
|------|------|
| 干支名 | 大字体居中（甲子、丙寅等） |
| 阴阳 | 小字体（阳/阴） |
| 主属性（天干五行） | 五行名称 + 对应颜色 |
| 卡牌底色 | 按主属性五行配色（浅色底 + 顶部主色条） |
| 当前评分 | 本季节评分（正绿负红），带一位小数 |
| 买入消耗 | 根据当前杠杆开关状态显示：普通 `消耗: 14气` / 杠杆 `消耗: 24气(杠)` |
| 选中状态 | 点击后橙色边框高亮（3px） |
| 牌堆为空 | 显示"牌堆已空"占位 |

#### 6. 操作按钮区

**布局：2×2 网格**

```
┌────────┐  ┌────────┐
│  买 入  │  │ 杠杆2x │
│  -14气  │  │  开/关  │
└────────┘  └────────┘
┌────────┐  ┌────────┐
│  卖 出  │  │  等 待  │
│  -3气   │  │  +17气  │
└────────┘  └────────┘
```

| 按钮 | 颜色 | 气变化 | 启用条件 | 禁用时 |
|------|------|--------|---------|--------|
| 买入 | 蓝色 #2196F3 | -N气（含杠杆加成） | 选中公共牌 + 手牌未满 + 气够 + 牌堆非空 | 置灰 |

**杠杆交互流程（三步操作）：**

```
Step 1: 点击公共牌池中的牌 → 牌高亮，买入按钮显示消耗
Step 2: 点击杠杆按钮 → 切换杠杆开关（开/关），买入消耗数字实时更新
Step 3: 点击买入按钮 → 执行买入（普通或杠杆），进入结算
```

| 状态 | 杠杆按钮外观 |
|------|-------------|
| 未选中公共牌 | 灰色，不可点击 |
| 选中公共牌，杠杆=1.0x | 灰色，显示 `杠杆 1.0x`，不可点击（提示无加成） |
| 选中公共牌，杠杆>1.0x，未开启 | 深灰 `杠杆 Nx 关`，可点击 |
| 选中公共牌，杠杆>1.0x，已开启 | 橙色 `杠杆 Nx 开`，可点击 |

**买入按钮气消耗动态显示：**

| 杠杆状态 | 显示内容 |
|---------|---------|
| 杠杆关闭 | `买入 -14气` |
| 杠杆开启 | `买入 -24气(杠)` |

### 动画与反馈规范

#### 动画优先级

| 优先级 | 动画 | 实现阶段 | 理由 |
|--------|------|---------|------|
| P0 | 卡牌买入/卖出飞行动画 | 第一批 | 最核心的游戏感来源，没有它像在操作表格 |
| P1 | 季节切换过渡动画 | 第一批 | 氛围感，强化季节博弈紧张感 |
| P2 | 分数/气条数字动画 | 第二批 | 锦上添花，让数字变化有存在感 |
| P3 | 结算画面过渡 | 第二批 | 回合节奏感 |
| P4 | 爆仓反馈 | 第二批 | 温和但明确的警告 |

#### P0：卡牌飞行动画

**买入动画（牌从牌池飞入手牌）：**

```
触发时机：玩家点击买入按钮后
持续时间：400ms
路径：从公共牌池位置 → 目标手牌空位（贝塞尔曲线）
效果：
  - 卡牌缩小到 0.8x → 飞行中保持 0.8x → 落地时弹回 1.0x（ overshoot bounce）
  - 飞行中卡牌 alpha 0.8
  - 落地时轻微弹跳（y 方向 -5px → 0，100ms）
  - 同时目标空位的虚线框消失
```

**卖出动画（牌从手牌飞出）：**

```
触发时机：玩家点击卖出按钮后
持续时间：350ms
路径：从手牌位置 → 向上飞出屏幕（y 方向 -100px）+ 缩小到 0
效果：
  - 卡牌向上飘出 + alpha 渐隐
  - 飞出时在原位置弹出收益文字（如 "+15.0 ▲"，绿色，200ms 后消失）
  - 杠杆卖出时收益文字为橙色
```

**实现要点：**
- 飞行中创建卡牌副本（sprite 或 container），不操作原卡牌
- 动画结束后销毁副本，刷新主 UI
- 使用 Phaser Tweens: `this.tweens.add({ targets, x, y, scaleX, scaleY, alpha, duration, ease })`
- ease: `'Back.easeOut'`（买入落地），`'Sine.easeIn'`（卖出飞出）

#### P1：季节切换动画

```
触发时机：executeTurn 检测到季节变更
持续时间：500ms（背景渐变）+ 300ms（图标弹入）
效果：
  1. 全屏覆盖一层新季节色（alpha 0→0.3→0，500ms）
  2. 季节图标从 scale 0 弹入到 scale 1.2 再回到 1.0（300ms，'Back.easeOut'）
  3. 同时显示文字提示 "春→夏 季节已切换！"（300ms 后淡出）
  4. 动画期间禁用操作按钮
```

#### P2：数字动画

**分数变化：**
```
总分变化时：scale 1.0→1.15→1.0（300ms，'Sine.easeInOut'）
盈亏数字出现时：alpha 0→1 + y 偏移 +5→0（200ms）
```

**气条动画：**
```
回复时：气条填充从旧宽度过渡到新宽度（300ms），同时气条颜色闪烁（alpha 1→0.5→1，200ms）
消耗时：同上，闪烁色为红色
```

#### P3：结算画面过渡

```
触发时机：操作执行后
效果：
  - 当前游戏 UI 整体 alpha 渐隐（150ms）
  - 结算画面从 y+30 滑入到 y=0（200ms，'Sine.easeOut'）
  - 继续按钮点击后反向过渡
```

#### P4：爆仓反馈（温和）

```
触发时机：checkMarginCall 返回 triggered=true
效果：
  1. 结算画面气量文字变红色 + 加粗
  2. 气量文字轻微震动（x 方向 ±3px，200ms，3次）
  3. 被强平的牌名逐条显示，红色文字 + ▲ 图标
  4. 不做全屏红闪、不做震屏、不做卡牌碎裂（太重）
```

## 边界情况

| 场景 | UI 处理 |
|------|---------|
| 分数为负数 | 红色显示，带 ▼ 标识 |
| 气为 0 | 气条全空 + 红色边框，买入/卖出按钮置灰 |
| 手牌全空 | 手牌区域显示 3 个空位，卖出按钮置灰 |
| 手牌全满 | 手牌区域无空位，买入按钮置灰 |
| 牌堆为空 | 公共牌池显示"牌堆已空"，买入按钮置灰 |
| 季节切换动画进行中 | 禁用操作按钮（500ms） |
| 杠杆=1.0x | 杠杆按钮置灰 + 显示 `1.0x`（提示无加成不划算） |

| 爆仓震动幅度 | 3px | 2px ~ 5px | 警告强度 |
| 气条颜色阈值 | 60%/30% | 50%/25% ~ 70%/40% | 低气警告时机 |
