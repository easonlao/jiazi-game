## 一致性检查

### 🔴 阻塞性问题

无。所有 GDD 之间在核心规则上没有发现直接矛盾。

### ⚠️ 警告

#### 1. 依赖方向不完整（多处）
- **涉及 GDD**: `system-hand-cards.md` 与 `system-qi-resource.md`
- **问题**: `system-hand-cards.md` 的依赖表中列出依赖 `#3 气资源`，但 `system-qi-resource.md` 的依赖表中没有反向列出 `system-hand-cards.md`。
- **建议**: 更新 `system-qi-resource.md` 的依赖表，补充"手牌管理（卖出/买入操作的气消耗检查）"。

#### 2. 杠杆气耗公式定义重复且不一致
- **涉及 GDD**: `system-leverage.md` 与 `system-qi-resource.md`
- **问题**: 两处都定义了持仓气耗公式：
  - `system-leverage.md`: `max(0.5, 1.5 + 0.4 × 评分) × 杠杆倍数`
  - `system-qi-resource.md`: `max(0.5, 1.5 + 0.4 × 当季评分) × 杠杆倍数`
  公式一致，但"评分"定义不统一 —— 一处写"评分"，一处写"当季评分"。在 MVP 阶段，买入评分与当季评分可能不同。
- **建议**: 统一术语，明确持仓气耗使用**当季评分**（即每回合重新计算的卡牌评分），而非买入时评分。在 `system-qi-resource.md` 中补充说明。

#### 3. 卖出操作的气回复时序不清晰
- **涉及 GDD**: `system-turn-flow.md` 与 `system-hand-cards.md`
- **问题**: 
  - `system-turn-flow.md` 的回合流程中，卖出操作描述为"即时回复 CS(8) 气"，但气回复步骤（第5步）在玩家操作**之前**执行。
  - 如果卖出的气回复在当回合生效，则玩家可以在同一回合用回复的气再买入，但回合流程规定每回合只能做一个操作。
- **建议**: 明确规则：卖出的气回复**在本回合生效**，但由于只能做一个操作，不能连续买卖。只需在规则中注明即可。

#### 4. 空亡牌系统缺失交叉引用
- **涉及 GDD**: `game-concept.md`（定义了空亡牌）与 `system-card-pool.md`（未提及）
- **问题**: `game-concept.md` 在 Alpha 阶段设计中定义了空亡牌（特殊牌），但 `system-card-pool.md` 没有任何关于特殊牌的结构预留。
- **建议**: 在 `system-card-pool.md` 的"扩展预留"部分添加空亡牌的处理框架。

## 游戏设计问题

### 🔴 阻塞性问题

无。核心游戏循环完整，MVP 定义清晰。

### ⚠️ 警告

#### 1. 缺少设计支柱文档
- **问题**: 项目没有 `design/gdd/game-pillars.md` 文件。`game-concept.md` 中定义了 4 个设计支柱和 3 个反支柱，但这些没有独立记录，也没有被其他 GDD 引用验证。
- **建议**: 创建 `game-pillars.md`，从中提取支柱定义，并让每个 GDD 在 Player Fantasy 部分显式映射到至少一个支柱。

#### 2. 冷启动体验未定义
- **涉及系统**: 回合流程、手牌管理、气资源
- **问题**: 游戏第一回合开始时，手牌为空，气 = 50，公共牌池有 2 张牌。玩家只能选择买入或等待。如果两张牌评分都很低（负分），买入会导致立即亏损。没有指导新手玩家的机制。
- **建议**: 考虑在 MVP 中加入简单的"教程提示"（第一次显示，可关闭），或者在第一个季节的评分表中确保前几回合出现的牌多为正评分。这不是设计缺陷，但在 Alpha 测试时需要验证。

## 跨系统场景问题

### 场景 1: 高杠杆持仓 + 季节切换
- **涉及系统**: 杠杆系统、季节循环、盈亏结算、气资源
- **步骤**:
  1. 玩家在季节末尾（杠杆 3.0x）买入一张当季牌
  2. 下回合季节切换到对立季节
  3. 持仓牌评分从 +4（当季）变为 -3（对立）
  4. 持仓收益 = 1.2 × (-3) × 3.0 = -10.8 分/回合
  5. 持仓气耗 = max(0.5, 1.5 + 0.4 × (-3)) × 3.0 = max(0.5, 0.3) × 3.0 = 0.5 × 3.0 = 1.5 气/回合
- **分析**: 持仓气耗公式在负评分时取 max(0.5, ...)，0.3 被替换为 0.5，导致负分牌反而比正分牌（假设评分 4 → 1.5+1.6=3.1，max=3.1）持有成本低得多。这是否符合设计意图？
- **建议**: 验证持仓气耗公式与评分的关系：负评分牌持有成本更低，可能鼓励玩家持有亏损牌等待反转？在调参时检查。

### 场景 2: 爆仓连锁
- **涉及系统**: 杠杆系统、气资源、手牌管理
- **步骤**:
  1. 玩家持有 3 张杠杆牌（各 2.0x）
  2. 持仓气耗每回合扣到气归零
  3. 触发爆仓：随机强平一张杠杆牌
  4. 强平后气仍为 0 → 继续强平第二张 → 直到无杠杆牌或气 > 0
- **分析**: 当前规则是"随机选择一张杠杆牌强制卖出"，如果多张牌，顺序随机。玩家无法控制哪张被强平，可能强平掉最有价值的牌。
- **建议**: 考虑设计为"强制卖出杠杆倍数最低的牌"（保护高价值投资）或让玩家选择（但会打断游戏节奏）。当前随机方案在 MVP 可接受，但需在 Alpha 测试中验证玩家感受。

### 场景 3: 牌堆为空 + 手牌为空
- **涉及系统**: 公共牌池、回合流程
- **步骤**:
  1. 60 张牌全部被买入或卖出
  2. 公共牌池无牌
  3. 手牌为空
  4. 玩家只能点击"等待"，每回合回复 17 气，但无法得分
  5. 剩余回合玩家无事可做，只能等到游戏结束
- **分析**: 极端情况，在正常游戏中不太会发生（60 张牌，每张买入并卖出需要两个操作），但如果玩家大量等待，可能出现。
- **建议**: 添加结束条件：当牌堆为空且手牌为空时，游戏提前结束。或在 UI 中添加"提前结束"按钮。
